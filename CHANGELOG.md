# Changelog

## [2.4] - 2026-05-20

### Глобальный баг фикс

Полный аудит кодовой базы, исправление критических и средних багов,
удаление нерабочих скриптов, стабилизация загрузчика.

### Исправления

**Критические:**
- **loader.py** — полностью переписан с нуля. Чистый код, надёжная загрузка
  скриптов, понятное логирование при старте, корректная очистка при ошибках
- **translator/main.py** — `from deep_translator` перенесён из верхнего уровня
  внутрь `register()`. Раньше отсутствие модуля крашило загрузку ВСЕХ
  скриптов через `exec_module()`
- **ai_chat/main.py** — удалён дубликат функции `_extract_username`,
  который перезаписывал первую версию и мог вызывать непредсказуемое поведение

**Важные:**
- **weather/main.py** — `asyncio.get_event_loop()` заменён на
  `asyncio.get_running_loop()` (deprecated в Python 3.10+)
- **translator/main.py** — тот же фикс `asyncio`
- **Все аддоны переводчика** (ru, de, zh, fr, uk, be) — фикс `asyncio`
- **config.py** — добавлено `os.makedirs()` для `scripts/` и
  `scripts_custom/`, чтобы директории создавались при первом запуске

**Мелкие:**
- **ai_chat/main.py** — удалён неиспользуемый импорт `MessageEntityType`
- **ai_chat/main.py** — удалён дубль `"проанализируй"` из списка ключевых слов

### Удаления

- `scripts/action_logger/` — удалён (ненужный overhead)
- `scripts/time_name/` — удалён (не используется)
- `scripts/whotgk/` — удалён (не используется)
- `scripts/yandex_music/` — удалён (не используется)
- `yandex-music` удалён из `requirements.txt`

### Изменения

- **.gitignore** — добавлены `*.pyc`, `ops/` для корректной работы
- **config.py** — добавлена `VERSION = "2.4"`
- **bot.py** — версия отображается при старте в логах
- **requirements.txt** — убраны неиспользуемые зависимости
